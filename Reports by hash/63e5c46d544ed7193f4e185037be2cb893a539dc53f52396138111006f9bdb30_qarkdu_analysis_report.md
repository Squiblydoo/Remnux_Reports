## 2. Classification

| **Confidence** | **HIGH** |

`qarkdu.exe` is a lean, single-purpose download-and-execute dropper whose sole function is to fetch and launch the StealC infostealer payload from a hardcoded C2 URL. The binary masquerades as a Windows system binary (svchost.exe) and carries a legitimate-looking EV code-signing certificate to evade SmartScreen and AV heuristics.

---

## 3. Capabilities

- **Identity masquerading**: VersionInfo cloned from legitimate Windows svchost.exe (T1036.001)
- **Fake EV code signing**: SSL.com 1-year EV cert for a UK company (T1553.002)
- **Base64 URL obfuscation**: C2 download URL stored as base64 in `.rdata`, decoded at runtime (T1027)
- **WinInet HTTP downloader**: Opens URL with `Mozilla/5.0` user agent, reads payload in 8KB chunks (T1071.001, T1105)
- **Temp file drop-and-exec**: Writes downloaded PE to `%TEMP%\scrXXXX.TMP` via `CreateFileA`+`WriteFile`, executes with `CreateProcessA(DETACHED_PROCESS)` (T1059)
- **File cleanup**: `DeleteFileA` on temp file after 2-second `Sleep` (T1070.004)
- **Anti-debug stubs**: `IsDebuggerPresent`, `IsProcessorFeaturePresent`, `RaiseException`, `UnhandledExceptionFilter` present in imports (T1622)
- **XOR loops**: 4 XOR-in-loop obfuscation routines detected (used in string/data processing)
- **Stack string construction**: 2 stack array initialization patterns (additional obfuscation)

## 6. Emulation Results

**Speakeasy pass 1 (generic runner)**: No IOCs — execution reached CRT init then halted at `kernel32.FlsGetValue2` (a Windows 10 21H1+ API not stubbed by speakeasy). The network download code was never reached.

**Speakeasy pass 2 (direct)**: Same result — `kernel32.FlsGetValue2` (unsupported API) caused emulation to abort before the download logic at `sub_401000`.

**Floss (deobfuscated strings)**: Confirmed partial C2 URL fragment `http://192.109.200.164/to/i` (consistent with static base64 decode).

Emulation was partially blocked by the unsupported FlsGetValue2 API stub. The complete attack chain was recovered via disassembly (see §4).

## 8. Analyst Notes

**StealC attribution**: ANY.RUN's sandbox engine independently identified the payload as StealC with maximum confidence. This sample is the dropper stage; the StealC binary itself would be `invoice20026556576888.exe` downloaded at runtime.

**EV certificate abuse**: The "ENGINEERING AND TECHNICAL PROCUREMENT SERVICES LTD" SSL.com EV cert (issued 2026-04-14, 6 days before the binary's build timestamp) follows the well-documented threat actor pattern of obtaining short-tenure EV certs to bypass SmartScreen trust checks. This cert serial (`4770710b7dd24ab39cd5547476fb165c`) has not appeared in previously analyzed samples in this workspace — no shared infrastructure confirmed.

**Filename deception**: `qarkdu.exe` is a randomized filename used for delivery; the binary internally identifies as `svchost.exe`. Victims who inspect the file properties may be deceived into believing it is a legitimate Windows binary.

**Sections**: The custom `.fptable` section (non-standard name) and high `.text` section entropy (6.57) are anomalies worth flagging in YARA rules.

**Recommended follow-up**:
1. Submit `invoice20026556576888.exe` (the StealC payload) for separate analysis if it can be retrieved
2. Block `192[.]109[.]200[.]164` at the perimeter; sinkhole or null-route at DNS level
3. Revoke/flag the SSL.com cert serial `4770710b7dd24ab39cd5547476fb165c` with the CA
4. Yara rule on the base64 URL string and "scr" temp prefix pattern
