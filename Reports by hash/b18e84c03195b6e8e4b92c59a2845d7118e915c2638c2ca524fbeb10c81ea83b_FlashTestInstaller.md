# FlashTestInstaller.exe ŌĆö Malware Analysis Report
FlashTestInstaller.exe ŌĆö Chrome Browser Preferences Hijacker

  Classification: MALICIOUS (HIGH confidence)
  What it is

  A PUA/browser hijacker disguised as a network speed test installer. The lure (FlashTest.exe) is a fully functional WPF speed test app. The real payload silently exfiltrates and tampers
  with Chrome browser settings.

  Core attack mechanism

  1. Chrome detection ŌĆö CbrDetectionService looks for chrome.exe via registry (HKLM\...\App Paths) and filesystem, recording the browser version. The string "chrome" is obfuscated as a
  char-array: DecodeChars(99,104,114,111,109,101,46,101,120,101).
  2. Server-gated activation ŌĆö Contacts https://colropl.com/fls for config. The prevf flag controls whether the hijack runs; the opt-out checkbox label is also server-supplied (can be
  made misleading).
  3. Chrome profile exfiltration + tampering ŌĆö For each Chrome profile:
    - Reads raw bytes of Secure Preferences + Preferences files
    - XOR-encrypts (key: Xt9kLm3nQp7wYz-f1829574063) and POSTs to https://api.colropl.com/ValidateFlash with the user's Windows SID and MachineGuid
    - Receives operator-controlled modified versions back
    - Overwrites Chrome profile files, kills Chrome, restarts it
  4. Telemetry ŌĆö Extensive install analytics (OS, Chrome version, user interaction, outcome) sent to https://colropl.com/fs. Uninstall reported via https://colropl.com/uti.

  Key IOCs

  - C2 domains: colropl.com, api.colropl.com
  - XOR key: Xt9kLm3nQp7wYz-f1829574063
  - Cert serial: 00a888cb01c4a97f105fda08f27c7bb2bc (Kartos Gale LTD, Tel Aviv IL)
  - Artifacts: %LOCALAPPDATA%\FlashTest\, cis.txt (persistent ID), temp_merge_*.tmp in Chrome profile dirs

  What's operator-gated / unknown

  The specific Chrome settings modified (search engine, extensions, NTP, etc.) are determined server-side and not visible in static analysis. An Updater.exe is referenced but not embedded
   ŌĆö likely downloaded in live deployments.

## 3. Classification

**Family:** Chrome Browser Preferences Hijacker (custom, no public family match)
**Category:** PUA / Browser Hijacker
**Confidence:** HIGH

**Reasoning:**
- The installer's primary hidden function (`BeginMergeOperation`) reads Chrome's `Preferences` and `Secure Preferences` JSON files, uploads the raw bytes to a remote C2 (`ValidateFlash`), receives operator-supplied modified versions back, and overwrites the originals ŌĆö silently changing browser settings (likely default search engine, extensions, or homepage).
- The lure (`FlashTest.exe`) is a functional WPF network speed test app that provides cover.
- Extensive anti-detection obfuscation: Chrome executable name encoded as char array `DecodeChars(99,104,114,111,109,101,46,101,120,101)` = `chrome.exe`; Chrome file paths received from C2 XOR-encrypted; fake future PE timestamp.
- Server-side operator gating (`prevf` flag): the hijack only proceeds if the C2 enables it for this victim.

## 4. Capabilities

### Installer (FlashTestInstaller.exe)
- **System fingerprinting:** Collects OS version/type, Windows `MachineGuid` as persistent user ID; Chrome browser version
- **Chrome detection:** `CbrDetectionService` checks `HKLM/HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\App Paths\chrome.exe` and filesystem paths for `chrome.exe`
- **Server config fetch:** Contacts `https://colropl.com/fls` on startup; receives: channel ID (`Ce_id`), source ID (`S_euid`), whether to deploy (`prevf`), browser flag (`D_ebr`), session ID, IOI key
- **Chrome profile hijack (conditional):**
  1. POSTs user ID to `https://api.colropl.com/StartFlash` ŌåÆ receives session token + XOR-encrypted Chrome path config
  2. Reads Chrome `Local State` to enumerate all profile directories
  3. For each profile: reads raw bytes of `Secure Preferences` + `Preferences`
  4. XOR-encrypts (key: `Xt9kLm3nQp7wYz-f1829574063`) + URL-encodes the data
  5. POSTs to `https://api.colropl.com/ValidateFlash` with `{FlashUser: SID, TestId: MachineGuid, DB1: SecurePrefs bytes, DB2: Prefs bytes}`
  6. Receives operator-modified file content back; decrypts; writes to Chrome profile with temp backups (`temp_merge_primary.tmp`, `temp_merge_secondary.tmp`)
  7. Kills Chrome ŌåÆ waits 10 seconds ŌåÆ restarts Chrome
  8. Verifies hijack success by checking for validation token in written file
- **Telemetry transmission:** Sends `AnlzData` to `https://colropl.com/fs` including: user ID, OS info, Chrome version, install outcome flags (`typop`, `spitz`), user interaction flags (checkbox state, close behavior), channel/source IDs
- **Merge status reporting:** POSTs `{IsValid, UserId, ErrorMsg}` to `https://api.colropl.com/FirstReportFe`
- **Merge metrics reporting:** POSTs `{IsRetry, SpStatus, TestUserId, ft_ver, ft_prof}` to `https://api.colropl.com/SecondReportTa`
- **Legitimate installer actions:** Extracts `FlashTest.exe`+`Uninstall.exe` to `%LOCALAPPDATA%\FlashTest\`, creates Start Menu shortcut, registers in Add/Remove Programs
- **WScript.Shell usage:** Used for creating shortcuts
- **XOR obfuscation:** `EncryptionHelper` (key: `Xt9kLm3nQp7wYz-f1829574063`) obfuscates all Chrome path data in transit and in C2 responses
- **Dynamic code loading:** `Assembly.Load` usage detected (2 hits)
- **Conditional execution:** Merge only runs if: (a) Chrome installed, (b) user did not uncheck the opt-out checkbox, (c) server `prevf != "false"`

### FlashTest.exe (lure app)
- Functional WPF network speed test: download (via `spd.flanativ.com/100mb`, `/1mb`), upload, ping, jitter
- IP/ISP geolocation via `spd.flanativ.com/cdn-cgi/trace`
- Saves test history to `history.xml`
- On uninstall (launched with `--uninstall-event` arg by Uninstaller): POSTs `MachineGuid` to `https://colropl.com/uti`

### Uninstall.exe
- Standard WPF uninstaller
- Kills running processes, removes registry entries, removes shortcuts, removes scheduled task (`FlashUpdater`)
- Launches `FlashTest.exe --uninstall-event` to notify C2 of removal

## 8. Analyst Notes

### What is confirmed (static analysis)
- The `ValidateFlash` endpoint receives the raw binary content of Chrome's `Secure Preferences` and `Preferences` files and returns modified replacements. The final Chrome settings change (search engine, extensions, etc.) is **operator-controlled server-side** ŌĆö not visible in this sample.
- The hijack is **conditional**: requires Chrome installed, user not opting out, and server enabling it (`prevf != "false"`). This is a soft opt-out mechanism intended to claim "consent."
- The opt-out checkbox is dynamically loaded (`FetchCheckboxTextAsync`) ŌĆö the checkbox text/label comes from the server, allowing operators to make it vague or misleading.

### Gaps requiring dynamic analysis
1. **What exactly is modified in Chrome Preferences/Secure Preferences** ŌĆö the specific fields changed (default search, extensions, NTP) are determined server-side and not visible without live C2 interaction.
2. **`baca.txt`** ŌĆö referenced in strings but not found in installer code paths examined; may be a legacy artifact or used in a separate code path.
3. **`Updater.exe`** ŌĆö referenced as a constant (`UpdaterExe = "Updater.exe"`) but not present in embedded resources; may be downloaded from C2 during dynamic execution if server sends a download URL via `D_ebr` config field.
4. **`SystemInfoService.IsValidUser()`** ŌĆö gates the merge operation; the conditions for "preferred user" are not fully analyzed.
5. **`GetTargetName(1/2/3)`** ŌĆö builds Chrome executable path from obfuscated components; args 1/2/3 likely decode to "Google"/"Chrome"/"chrome.exe" but not confirmed.

### Alternative hypotheses
- **Lower confidence (medium):** The modification to Chrome `Preferences` could be limited to installing a legitimate affiliate browser extension (rather than full search hijacking), which would be PUA rather than malware. However, the upload of raw preference file bytes to a remote server is inherently a privacy violation regardless of the downstream modification.
- The `Updater.exe` constant and `ShouldDownloadUpdater()` logic suggest a second-stage payload may be downloaded in some configurations not present here.

### Threat Assessment
**MALICIOUS** ŌĆö This sample covertly uploads the contents of Chrome browser profile files to a remote operator server and writes back modified versions, constituting unauthorized access to and modification of browser credentials/settings storage. The EV code signing certificate ("Kartos Gale LTD") and legitimate-looking lure app are deliberate deception. The server-gated nature provides operator control and deniability.

**Recommended response:**
1. Block `colropl.com`, `api.colropl.com`, `flanativ.com`, `spd.flanativ.com` at network perimeter
2. Check Chrome's `Secure Preferences` and `Preferences` for unauthorized modification (compare against known-good baseline)
3. Reset Chrome to defaults / clear all extensions
4. Remove `%LOCALAPPDATA%\FlashTest\` directory
5. Delete any `temp_merge_*.tmp` files in Chrome profile directories
