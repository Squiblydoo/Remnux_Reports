## 2. Classification

**Family**: Korean Screen Locker / File-Targeting Ransomware (custom/unnamed; "DROPPER2" internal name)  
**Confidence**: HIGH  
**KesaKode** (low confidence): WinBeast 56%, Boramae 53%, TFlower 50%

**Reasoning**: The PDB path explicitly names this `DROPPER2.pdb`. The binary:
- Is signed with a Korean company EV cert (ToCSG Co.,Ltd, Seoul)
- Implements the **SEED block cipher** (ISO/IEC 18033-3, Korean national standard) for file encryption
- Statically links libcurl+OpenSSL to communicate with a Korean C2 (`h-sec.co.kr`)
- Specifically targets **HWP (Hangul Word Processor)** files exclusive to South Korea alongside standard Office/PDF formats
- Contains a hardcoded **victim email** (`rob.smith@qcells.com`) identifying this as a **targeted delivery** against **Qcells** (Korean/German solar energy company, Hanwha group subsidiary)
- Downloads a complete locker UI configuration from the C2, including ransom message text, countdown timer, background image, and decryption password
- Reports encrypted file statistics (count + KB by file type including HWP) back to C2

## 3. Capabilities

- **File encryption**: SEED block cipher (Korean national standard) targeting .DOC, .DOCX, .XLS, .XLSX, .XLSM, .PPT, .PPTX, .PDF, .HWP
- **Drive enumeration**: Finds DRIVE_FIXED volumes via `GetDriveType` / bitwise check; logs `drive %c  : DRIVE_FIXED`
- **C2 check-in**: Downloads locker config via HTTPS POST to `/v2/scenario/svc/locker/config.do`
- **Locker UI**: Displays full-screen lock window with configurable background image, countdown timer, ransom message text, and payment URL (all served from C2)
- **Telemetry**: Reports encrypted file counts/sizes by type (docFCnt/docFKb, xlsFCnt/xlsFKb, pptFCnt/pptFKb, pdfFCnt/pdfFKb, **hwpFCnt/hwpFKb**) to C2
- **Password gating**: Decryption password is delivered by C2 (payment-gated); field `password` in config response
- **Base64 encoding**: Used to encode API request bodies
- **Multiple cipher support**: SEED (file encryption), AES (key schedule confirmed via S-box tables), ChaCha20/Salsa20 (`expand 32-byte k` constant present from OpenSSL)
- **Hardware fingerprinting**: YARA hit on `FingerprintHardware`
- **Shell launch**: YARA hit on `RunShell`
- **HTTP form upload**: YARA hit on `PostHttpForm`
- **Import-by-hash**: One IAT resolution by hash (`ImportByHash` anomaly)
- **Anti-analysis**: 249 XOR-in-loop instances; control-flow cross-section jump; stack array construction

## 6. Emulation Results

**Pass 1 (generic speakeasy runner, x86)**: Completed with 0 IOCs. Binary ran through libcurl/OpenSSL initialization (CriticalSection setup, HeapAlloc) and exited normally via `ExitProcess(0)`. No network connections — binary requires live C2 response to proceed.

**Pass 2 (plain speakeasy, x86)**: Completed. Only observed:
- File access: `D:\opensource\openssl-dist-1.1.0f-vs2017\openssl-x86-static-release-vs2017\ssl/openssl.cnf` (OpenSSL init, hardcoded build path)
- Binary resolved `mscoree.CorExitProcess` → called `ExitProcess(0)`

**Conclusion**: Emulation blocked by network dependency. The binary silently exits if the C2 handshake (`locker/config.do`) fails. No further emulation passes applied (Passes 3–4 not warranted; the decrypt functions are not the bottleneck — the C2 communication gate is).

## 8. Analyst Notes

### Summary
This is a **custom Korean screen locker / ransomware dropper** explicitly targeted at an individual (`rob.smith@qcells.com`) at **Qcells** (Hanwha Q Cells, a South Korean solar panel manufacturer). The binary is signed with a fresh 1-year Korean code signing cert (ToCSG Co.,Ltd, DigiCert), implements the Korean national SEED cipher, and specifically targets HWP files used by Korean businesses.

The C2 backend (`h-sec.co.kr:5500`) provides a fully managed locker experience: background image, countdown timer, ransom message text, and payment URLs are all operator-controlled and delivered at runtime. The decryption password is also C2-held, making offline decryption impossible without the operator's cooperation.

### Gaps / Follow-up

1. **C2 hostname `h-sec.co.kr`**: Check WHOIS registration date and hosting history. The format `h-sec` may imply "hacking-security" or be an actor alias. IP `203.233.82.247` should be geolocated and checked for prior abuse history.

2. **Cert abuse**: ToCSG Co.,Ltd cert serial `07577c360aa6955afbc0a73c1c02ee67` should be reported to DigiCert for revocation. The build date (2023-01-19) predates the cert issuance (2025-09-05), suggesting this sample may have been re-signed for the current campaign.

3. **Campaign codes** (`3866f3d9f9144568ba634fe7b4c0c778`, `859512a5f1084867aad3c7390c488a82`, `8b9747863e30430498dfd4b250c72799`): These appear to be scenario/project/content/domain codes used to identify the campaign in the C2 backend. The C2 uses a structured API suggesting a multi-tenant or multi-campaign RaaS-like platform.

4. **Attribution**: KesaKode low-confidence matches (WinBeast, Boramae, TFlower) should not be treated as attribution. "Boramae" is Korean for "young hawk" which may be coincidental. No strong family match to known public families.

5. **Qcells notification**: This binary contains the hardcoded email `rob.smith@qcells.com`. The Qcells security team should be informed of this targeted attack. The binary was delivered under a randomized filename (`favccml.exe`) suggesting spear-phishing delivery.

6. **Additional samples**: The PDB suffix `BSD2/dropper` (DROPPER2) implies at least a DROPPER1 exists. Other samples with the same cert serial or similar C2 API paths should be hunted in VirusTotal/MWDB.
