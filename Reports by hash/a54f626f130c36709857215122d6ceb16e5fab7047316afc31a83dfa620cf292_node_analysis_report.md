## 2. Classification

**Family:** CryptoChanger Wrapper — Pay-Per-Install (PPI) / Affiliate Dropper  
**Confidence:** High  
**Threat Type:** Downloader/Dropper with affiliate tracking infrastructure  

**Reasoning:**  
The overlay (95,640 bytes of V8 bytecode, source `C:\snapshot\wrapper\index.js`) contains a JavaScript wrapper whose sole purpose is to:
1. Read its own filename to extract a campaign token using the regex `CryptoChanger-([a-f0-9]{8})`.
2. Call the C2 (`666777228[.]com`) to fetch UTM/affiliate tracking parameters for the token.
3. Download and execute an installer payload from the C2.

The binary is signed with a legitimate code-signing certificate issued to a Finnish company ("Soft Insanity Oy") to bypass SmartScreen. The VersionInfo is spoofed to impersonate Node.js. This is a classic PPI / affiliate install wrapper architecture where the 8-character hex token in the filename identifies the distributing affiliate partner and campaign.

## 4. Attack Chain

```
Stage 1: Distribution
  Victim downloads/receives node.exe renamed as CryptoChanger-<8hexchars>.exe
  (delivered via affiliate network, social engineering, or malvertising)

Stage 2: Execution & Token Extraction
  node.exe launches → pkg bootstrap loads V8 bytecode from overlay
  JS: process.execPath + path.basename → gets own filename
  JS: match(/CryptoChanger-([a-f0-9]{8})/) → extracts affiliate token

Stage 3: C2 Beacon (UTM Tracking)
  HTTPS GET https://666777228[.]com/api/utm/<token>
  C2 responds with UTM campaign metadata (JSON: {success: bool, utm: {...}})
  Purpose: affiliate attribution / campaign tracking

Stage 4: Payload Download
  HTTPS request to https://666777228[.]com/ccw
    OR https://666777228[.]com/api/download-installer
  Receives installer binary

Stage 5: Execution
  child_process.spawn → executes downloaded installer
  (payload unknown; likely adware, PUP, or crypto-themed software)
```

## 7. Sandbox Results

**ANY.RUN:** Task ID `f9b31e73-4e57-4dcf-88a8-e1d4ef2e7ad9`  
**Score:** 35/100 — "No threats detected"  
**Tags:** None  
**Public URL:** `https://app.any.run/tasks/f9b31e73-4e57-4dcf-88a8-e1d4ef2e7ad9`

**Key behavioral observation:** The sandbox executed the binary as `node.exe`, which does **not** match the regex `CryptoChanger-([a-f0-9]{8})`. As a result, the JavaScript wrapper exits early ("No token found in filename") and no payload is downloaded. This is the primary **sandbox evasion mechanism** — the dropper is inert unless named correctly.

**DNS confirmed:** Despite early exit, a DNS query to `666777228[.]com` was observed, confirming the domain is contacted at runtime. No malicious HTTP/HTTPS traffic to `666777228[.]com` was captured (connection not completed). All other network activity was Windows-internal (OCSP, CRL, telemetry).

## 8. Analyst Notes

### What is confirmed
- The binary is a **legitimate Node.js 18.5.0 runtime** (built from source, PDB path matches a GitHub Actions runner environment) combined with a malicious JavaScript payload via Vercel's `pkg` bundler.
- The JavaScript overlay (~6KB source, compiled to V8 bytecode) contains the complete dropper logic — no native-code obfuscation is needed because the JS is pre-compiled to bytecode.
- The domain `666777228[.]com` is the sole C2 infrastructure. The number-only domain name is a common low-cost PPI network tactic.
- The token-in-filename architecture is consistent with **Pay-Per-Install (PPI) affiliate networks** where each distributor embeds a unique campaign token in the filename to track install attribution and commission payments.

### Cross-reference: npp.8.8.1.Installer.x64.exe (SHA256: `dc31cf21...`)

**Identical certificate serial:** `0db0cb5c25bc7af916078b05a5902aed` — "Soft Insanity Oy" / GoGetSSL, 2025-11-14→2026-11-13.

Both samples use the same code-signing certificate. The earlier Notepad++ installer was assessed as a re-signed legitimate installer; however, in light of `node.exe` being a confirmed malicious CryptoChanger dropper signed by the same entity, that assessment should be revisited. The "Soft Insanity Oy" certificate is being used to sign PPI-network payloads, and the NPP installer may be a secondary distribution vehicle for the same campaign (possibly a legitimate+trojanized bundle where the malicious component evaded analysis due to a different filename check).

**Recommended action:** Re-analyze `npp.8.8.1.Installer.x64.exe` with the filename renamed to match the `CryptoChanger-([a-f0-9]{8}).exe` pattern, or extract and examine its NSIS payload more thoroughly for a bundled pkg-wrapped executable.

### Gaps and follow-up
- **Payload unknown:** The actual installer downloaded from `/ccw` was not retrieved. Recommend live sandbox execution (ANY.RUN result pending) or controlled network detonation to capture the Stage 5 payload.
- **V8 bytecode decompilation:** The original `index.js` source (~5,954 bytes) could be recovered using a V8 bytecode decompiler (e.g., `v8-bytecode-to-js`, `bytenode` reverse tooling) to confirm the full logic and any additional obfuscation.
- **Certificate abuse:** "Soft Insanity Oy" should be reported to GoGetSSL for certificate revocation.
- **Attribution:** The `CryptoChanger` product name and `666777228[.]com` domain suggest a cryptocurrency-themed PPI network (possibly distributing clipboard hijackers, mining software, or fake crypto exchange tools). No prior attribution found.
